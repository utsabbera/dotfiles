# Development Preferences

## Code Style

- Write self-explanatory code; STRICTLY avoid unnecessary inline explanatory comments - meaningful docstrings are fine to have
- No emojis in code or documentation
- Prefer clean and simple solutions over complex ones

## Standards & Best Practices

- Always use latest versions of tools and tech stack
- Follow industry standards and best practices
- Consider all possibilities and cross-question assumptions

## Git Workflow

- Exclude Claude attribution lines from commit messages
